# Workshop C (60 minutes): Content sprint (ship a PR)

## Outcome

Participants ship a small, high-quality contribution (typo fix, troubleshooting note, translation stub, or a short tutorial improvement).

## Agenda

### 0:00–0:10 — Pick a task

- Choose a beginner pain point
- Find a doc page to improve

### 0:10–0:40 — Write + cite

- Add or improve a section
- Add official references (docs/blog/GitHub)

### 0:40–0:55 — PR polish

- Ensure no secrets
- Run website build if changing site content (`cd website && npm run build`)

### 0:55–1:00 — Celebrate + next sprint

- Collect links to PRs
- Plan next sprint theme (providers / developers / tokenomics / community)
