# Translation & localization guide

## Goal

Make it easy for the community to translate learning content while keeping technical accuracy.

## What to translate first

1. `docs/00-introduction/*`
2. `docs/10-providers/*`
3. `docs/20-developers/instant-deploy-hub.md`
4. `community/meetups/MEETUP_IN_A_BOX.md`

## Workflow (simple)

- Copy an existing page and create a language-suffixed variant, e.g.:
  - `overview.md` → `overview-es.md`
- Keep the **official reference links** unchanged.
- Add a short note at the top:
  - “Translated from English, last synced on YYYY-MM-DD.”

## Consistency rules

- Don’t translate product names (Acurast, Hub, Processor Lite/Core).
- Keep code blocks unchanged.
- If you translate terms, add them to a glossary list (optional file per language).

## Review checklist

- Technical steps match the official docs
- No secrets or sensitive screenshots
- Links work
- Clear, simple language
