# Moderation & support playbook

## Goals

- Keep channels welcoming, safe, and useful
- Help users reach “first wins” quickly
- Prevent secret leakage

## Triage categories

- **Provider onboarding** (Lite/Core setup, Hub connection)
- **Developer onboarding** (Hub deploy, CLI deploy, runtime APIs)
- **Tokenomics/staking** (conceptual questions, official links)
- **Community/ambassador** (events, content, contributions)

## Standard operating procedure

1. **Safety first**: ask the user to redact secrets; remove messages with mnemonics/private keys.
2. **Clarify basics**: device/OS, network, which guide followed, what exact step failed.
3. **Link official source**: provide the relevant official doc/blog link.
4. **Offer next steps**: 1–3 targeted actions; avoid long troubleshooting trees.
5. **Escalate**: if unclear or looks like a bug, request logs (redacted) and open an issue.

## “No secrets” enforcement

Remove/ask to delete:

- seed phrases / mnemonics
- private keys
- API keys / tokens
- wallet screenshots containing sensitive data
