# Content calendar (starter)

This calendar is designed to steadily grow the curriculum with a predictable cadence.

## Weekly cadence (suggested)

- **Monday**: pick one “beginner win” to improve (docs or tutorial)
- **Wednesday**: ship one asset (diagram/script/template)
- **Friday**: publish an update post + share in community channels

## 6-week starter plan

### Week 1 — Fundamentals

- Improve `docs/00-introduction/*`
- Diagram: Acurast overview
- Video: Acurast in 5 minutes

### Week 2 — Providers

- Lite/Core troubleshooting expansion
- Provider setup video
- Meetup kit run-through

### Week 3 — Hub “instant deploy”

- Workshop A runbook refinement
- “Deployment lifecycle” diagram
- Short video clip script

### Week 4 — CLI deploy + env vars

- Expand CLI lab
- Add “secrets” tutorial
- Add common error catalog

### Week 5 — Runtime APIs

- Add examples for HTTP + env + signing
- Add advanced “oracle” case study outline

### Week 6 — Tokenomics + Staked compute

- Add explainer diagrams and a short video script
- Host a community Q&A session

## Release checklist (monthly)

- Ensure website builds
- Audit links to official sources
- Tag a release in GitHub (optional)
