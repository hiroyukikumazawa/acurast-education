# Slide deck outline (Acurast meetup/workshop)

## Deck goal

Help attendees understand Acurast, pick a path (provider or developer), and complete a first win.

## Suggested 10-slide structure

1. **Title**: “Acurast: decentralized compute powered by phones”
2. **Why it matters**: edge compute, decentralization, community-owned infra
3. **Roles**: providers, developers, users
4. **Processors**: Lite vs Core (high-level requirements)
5. **Hub**: what it is and what you do there
6. **Demo plan**: Instant deploy (Hub) OR CLI deploy
7. **Live demo**: (screenshare)
8. **Troubleshooting**: common failure modes + fixes
9. **Next steps**: provider path + developer path + Cloud Rebellion
10. **Contribute**: PRs, issues, community kit
