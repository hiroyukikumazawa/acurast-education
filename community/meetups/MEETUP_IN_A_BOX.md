# Meetup in a Box (Acurast Education)

## Goal

Run a high-quality Acurast meetup with a repeatable format, beginner-friendly demos, and clear next steps.

## Suggested formats

- **Intro meetup (60 min)**: fundamentals + provider onboarding + Q&A
- **Builder meetup (90 min)**: instant deploy + CLI deploy walkthrough + lab time
- **Community creator meetup (60 min)**: how to make tutorials/videos + content sprint

## Pre-event checklist (1–7 days before)

- **Venue/stream**
  - Confirm venue capacity and Wi‑Fi quality (or streaming setup)
  - Test screen sharing and audio
- **Materials**
  - Print/share QR links: docs, hub, this repo, Discord/community links
  - Prepare “Start here” slide and end-of-event CTA slide
- **Demo readiness**
  - If doing Hub instant deploy: have Hub open and the official steps ready
  - If doing CLI demo: ensure Node/npm works locally and `.env` is never shown on-screen
- **Safety**
  - Remind attendees: never share mnemonics/private keys; no screenshots with secrets

## Run-of-show (Intro meetup, 60 minutes)

### 0:00–0:05 — Welcome

- What is Acurast (one sentence)
- What attendees will learn today

### 0:05–0:20 — Fundamentals

- Roles: providers, developers, token holders/community
- Lite vs Core (high-level)

### 0:20–0:40 — Demo: instant deploy via Hub

Follow the official guide:
`https://docs.acurast.com/developers/deploy-first-app-console`

Key verification step:
`https://<processor-address>.acu.run`

### 0:40–0:55 — How to get started (paths)

- Providers: `https://docs.acurast.com/acurast-processors/`
- Developers: `https://docs.acurast.com/developers/deploy-first-app/`
- Cloud Rebellion: `https://rebellion.acurast.com/`

### 0:55–1:00 — Close + feedback

- Share repo link and contribution path
- Collect feedback (template below)

## Feedback form template (copy/paste)

- What was the most useful part?
- What was confusing?
- What do you want next: Provider / Developer / Tokenomics / Staked compute / Creator kit?
- Your experience level (new / intermediate / advanced)
- Are you interested in helping create content? (yes/no)
